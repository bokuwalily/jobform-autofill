# 日本の新卒エントリーフォーム パターン調査メモ

この拡張が対応すべきフォームの型を整理する。企業ごとに使う採用管理システム（ATS）が違い、レイアウト・項目・分割の仕方が大きく異なる。

## 主要 ATS / フォームベンダー

| ベンダー | 例ドメイン | レイアウト傾向 | 特徴 |
|---|---|---|---|
| axol（アクセスヒューマネクスト） | axol.jp | div ブロック（ラベルdiv→入力div） | 郵便/電話は分割box、自宅+携帯併記、休暇中連絡先、学校コード検索 |
| e2r | e2r.jp | table（th/td） | 生年月日は3テキスト、姓/名・カナ分割、市区郡/番地/建物が別欄、海外用の別box併設 |
| i-web（ヒューマネクスト） | リクルートサイト系 | table寄り | 学校マスタ検索が多い |
| マイナビ / リクナビ | 各社 | 独自 | 統一プロフィールから流用、独自バリデーション |
| HRMOS / sonar ATS / herp | 各社 | モダンなdiv/React | name属性が機械的、ラベルは別要素 |

## 頻出フィールドと型

- **氏名**: 姓/名の2分割（漢字・カナ別）。行ラベル「氏名」＋各box隣に「姓」「名」。カナは全角/半角/ひらがな指定あり。
- **生年月日**: ①3つのselect（年/月/日） ②3つのtextに「年/月/日」が後置 ③input[type=date]。
- **性別**: radio（男性/女性/回答しない・その他）。value がコード値のこともある。
- **郵便番号**: 3桁-4桁の2分割 box（+「郵便番号検索」ボタン、+海外用の単独box）。
- **住所**: 都道府県(select) / 市区郡町村(text) / 町域・番地(text) / 建物・部屋番号(text)。単一「住所」欄の場合もある。
- **電話**: 3分割 box。自宅電話と携帯電話を両方持つフォームが多く「いずれか必須」。携帯のみ入力が無難。
- **メール**: 本体＋確認用の2box。さらに「メールアドレス2（別アドレス）」を持つ場合があり、これは主アドレスの複製では**ない**。
- **学校**: 学校区分(大学/大学院 radio) / 設置区分(国立/公立/私立 radio) / 学校名（コード検索 or text） / 学校名の頭文字（全角カナ1字） / 学部 / 学科 / コース名 / 文理区分 / 系統(専攻系統 select) / 卒業(予定)年月 / 卒業見込み区分 / 現在の学年。
- **入学**: 大学入学年月。高校名・高校入学/卒業年を聞く企業もある。
- **進路**: 就職(入社)希望年月。
- **休暇中の連絡先/帰省先**: 「現在の連絡先と同じ」チェックで省略できることが多い。

## 設計上の含意（この拡張の方針）

1. **レイアウト非依存のラベル抽出**: th だけに頼らず、祖先を遡って「直前のラベル/見出し要素」を拾う（table/div/dl 全対応）。← v0.4 で実装。
2. **グループは上書き**: 電話/郵便/日付は前回の誤入力(stale)を消して入れ直す。単体欄は空のときだけ。
3. **自宅 vs 携帯**: 区別して携帯優先。自宅に紛れた携帯番号は消す。
4. **メール2は別物**: profile.email2 がある時だけ。確認用は本体と同値。
5. **検索ボタン式（学校コード）/頭文字カナ**: 構造的に自動化困難 → 手動。将来は「学校検索クリック→候補一致選択」を検討。
6. **診断ログ**: 分類できない欄を console に出力（`[就活autofill] 未分類フィールド`）。新フォームで漏れたらこれを報告→規則追加。

## 既知の未対応 / 今後

- 和暦（令和/平成）の生年月日。
- 証明写真アップロード、ガクチカ等の長文流し込み。
- React/Vue の厳格な制御（一部 setter 経由で反映されないケース）。
